The failure may come from a flag, a Unicode representation, the API used to consume the expression, or the mutable state of a shared <code>RegExp</code> object.</p> <p>That is why “try another pattern” is often a poor debugging strategy. A better approach is to turn the bug into a small test matrix.</p> <p>The matrix has four useful axes:</p> <div class="table-wrapper-paragraph"><table> <thead> <tr> <th>Axis</th> <th>Questions to <code>exec</code>, <code>matchAll</code>, or <code>replace</code>?</td> </tr> <tr> <td>Expected result and state</td> <td>Which matches, groups, replacement text, and <code>lastIndex</code> values
