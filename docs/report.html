<h2> จาก chatbot ธรรมดา สู่ AI ที่ทำงานแทนเราได้ — เราเติมอะไรเข้าไปบ้าง? </h2> <p>เวลาเราใช้ ChatGPT หรือ Claude ครั้งแรก — เราพิมพ์คำถาม ได้คำตอบ จบ</p> <p>นั่นคือ <strong>LLM (Large Language Model)</strong> ในรูปแบบพื้นฐานที่สุด: ข้อความเข้า → ข้อความออก</p> <p>แต่อยู่ ๆ มาวันหนึ่งเราเห็น AI ที่:</p> <ul> <li>เข้าเว็บหาข้อมูลให้</li> <li>อ่านไฟล์ในเครื่องเรา</li> <li>รันคำสั่ง terminal</li> <li>เขียนโค้ด สร้างไฟล์ รัน test</li> <li>ทำงานต่อเนื่องเป็นสิบนาทีโดยไม่ต้องคอยบอกทุกขั้นตอน</li> </ul> <p><strong>เราเติมอะไรลงไปบ้าง?</strong> — จาก chatbot ธรรมดากลายเป็น agent ได้ยังไง?</p> <h2> 🧱 เริ่มจาก LLM เปล่า ๆ </h2> <p>LLM คือ "เครื่องทำนายคำถัดไป" — มันเห็นข้อความก่อนหน้า แล้วเดาว่าคำต่อไปควรเป็นอะไร<br /> </p> <div class="highlight js-code-highlight"> <pre class="highlight plaintext"><code>[ผู้ใช้ถาม] → [LLM] → [คำตอบ] </code></pre> </div> <p>สิ่งที่ LLM <strong>ทำไม่ได้</strong> ในสภาพนี้:</p> <ul> <li>จำว่าเมื่อวานเราคุยอะไรกัน (ไม่มี memory)</li> <li>ค้นหาข้อมูลในเน็ต (ไม่มี tool)</li> <li>รู้วันที่วันนี้ (ถูก train ด้วยข้อมูลเก่า)</li> <li>ทำอะไรบนเครื่องเรา (ไม่มีสิทธิ์)</li> </ul> <p>มันคือสมองที่เก่ง — แต่ไม่มีแขน ไม่มีขา ไม่มีความจำ</p> <h2> ➕ Layer 1: System Prompt — ใส่ "บทบาท" และ "กติกา" </h2> <p>สิ่งแรกที่เราเติม: <strong>system prompt</strong> — ข้อความที่บอก LLM ว่า "คุณเป็นใคร" และ "คุณทำอะไรได้บ้าง"<br /> </p> <div class="highlight js-code-highlight"> <pre class="highlight plaintext"><code>คุณคือ AI ผู้ช่วยชื่อ Hermes คุณทำงานบนเครื่อง Linux ของผู้ใช้ คุณตอบเป็นภาษาไทย อ่านง่าย เป็นกันเอง </code></pre> </div> <p>System prompt คือ <strong>"job description"</strong> ของ AI — มันเปลี่ยน LLM จาก "เครื่องทำนายคำ" เป็น "ผู้ช่วยที่มีตัวตน"</p> <h2> ➕ Layer 2: Tool Calling — ใส่ "แขน" และ "ขา" </h2> <p>LLM เก่งเรื่องภาษา แต่มันกดเครื่องคิดเลขไม่เป็น — เราก็เลยให้ <strong>เครื่องมือ (tools)</strong> กับมัน</p> <p>วิธีทำงาน:<br /> </p> <div class="highlight js-code-highlight"> <pre class="highlight plaintext"><code>1. ผู้ใช้: "เช็คสภาพอากาศกรุงเทพวันนี้" 2. LLM คิด: "ฉันต้องเรียกฟังก์ชัน get_weather(city='Bangkok')" 3. ระบบ: เรียก get_weather() จริง → ได้ผลลัพธ์ { temp: 35, humidity: 70 } 4. ระบบ: ส่งผลลัพธ์กลับไปให้ LLM 5. LLM: "วันนี้กรุงเทพ 35 องศา ความชื้น 70% ครับ" </code></pre> </div> <p><strong>Tools ที่เราให้ AI ได้:</strong></p> <div class="table-wrapper-paragraph"><table> <thead> <tr> <th>Tool</th> <th>ตัวอย่าง</th> <th>LLM ทำอะไร</th> </tr> </thead> <tbody> <tr> <td>Web search</td> <td><code>search("ราคาทองวันนี้")</code></td> <td>ค้นเน็ต</td> </tr> <tr> <td>File read</td> <td><code>read_file("config.yaml")</code></td> <td>อ่านไฟล์</td> </tr> <tr> <td>Terminal</td> <td><code>terminal("git status")</code></td> <td>รันคำสั่ง</td> </tr> <tr> <td>Database</td> <td><code>query("SELECT * FROM users")</code></td> <td>ถาม database</td> </tr> </tbody> </table></div> <p><strong>สิ่งที่เปลี่ยน:</strong> LLM ไม่ได้แค่ "พูด" แล้ว — มัน "ทำ" ได้แล้ว</p> <h2> ➕ Layer 3: Loop — ใส่ "ความต่อเนื่อง" </h2> <p>tool ครั้งเดียวอาจไม่พอ — LLM อาจต้องเรียก tool หลายรอบเพื่อทำงานชิ้นหนึ่งให้เสร็จ<br /> </p> <div class="highlight js-code-highlight"> <pre class="highlight plaintext"><code>1. ผู้ใช้: "สร้างไฟล์ README.md ให้ project นี้" 2. LLM → tool: write_file("README.md") ← เขียน! 6. <h2> シンプルなチャットボットから、私たちのために働くAIへ ― 何を追加すれば良いのでしょうか？ </h2> <p>ChatGPTやClaudeを初めて使った頃は、質問を入力して回答を得るだけでした。</p> <p>これは、<strong>LLM（大規模言語モデル）</strong>の最も基本的な形態でした。つまり、メッセージ入力→メッセージ出力です。</p> <p>しかしある日、次のようなAIが登場しました。</p> <ul><li>ウェブサイトをスキャンして情報を検索する</li> <li>コンピューター上のファイルを読み込む</li> <li>ターミナルコマンドを実行する</li> <li>コードを記述し、ファイルを作成し、テストを実行する</li> <li>継続的な指示を必要とせずに、何万分も連続して動作する</li>

要約

基本的なLLMは入力テキストに対して出力テキストを返すだけだが、System Prompt、Tool Calling、Loop、Memory、Reasoning、Skills、Multi-Agentなど7つのレイヤーを積み重ねることで、Webを検索したりファイルを操作し、複数ステップの作業を自動実行できるAIエージェントに進化させることができる。 LLMに「仕事内容」「ツール」「記憶」「思考プロセス」「経験」「チーム」を段階的に追加していくことで、単なる質問応答システムから自立的に動作するエージェントへと変わっていく。 結果として、Agentic AIは LLM本体の知能が優れているのではなく、実行可能な「ツール」と継続的に「行動」できる設計が本質的な違いであり、MCP、RAG、ACPなどの標準プロトコルはこの基本構造の上に成り立っている。
